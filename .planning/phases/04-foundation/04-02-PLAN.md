  <action>
    Create `src/auth.rs`:
    - `validate_timestamp(timestamp_ms: u64) -> Result<()>`: check drift <= 5 minutes (300_000ms)
    - `auth_headers(signer: &LocalSigner, address: &Address) -> Result<HashMap<String, String>>`:
      1. Get current timestamp ms
      2. Validate timestamp
      3. Build message: `ClawDuel:auth:<address_lowercase>:<timestamp>`
      4. Sign with `signer.sign_message(message.as_bytes()).await` (alloy EIP-191)
      5. Return headers: `X-Agent-Address`, `X-Agent-Signature`, `X-Agent-Timestamp`, `Content-Type: application/json`

    Create `src/http.rs`:
    - `HttpClient` struct holding: `reqwest::Client` (with 30s timeout), `base_url: String`, `signer: LocalSigner`, `address: Address`
    - Constructor validates base_url with `security::validate_backend_url()`
    - `pub async fn get(&self, path: &str) -> Result<serde_json::Value>`:
      1. Build full URL, inject auth headers
      2. Send GET with timeout
      3. On 4xx/5xx, redact error body with `security::redact_secrets()`
      4. Return parsed JSON
    - `pub async fn post(&self, path: &str, body: &impl Serialize) -> Result<(u16, serde_json::Value)>`:
      1. Serialize body to string
      2. Call `security::assert_no_secret_leak(&serialized, &self.private_key_hex)`
      3. Build full URL, inject auth headers
      4. Send POST with timeout
      5. On 4xx/5xx, redact error body
      6. Return (status_code, parsed_json)
    - `pub async fn request(&self, method: Method, path: &str, body: Option<&impl Serialize>) -> Result<(u16, serde_json::Value)>`:
      Generic method for DELETE etc (used by dequeue command later).
    - Timeout: `reqwest::Client::builder().timeout(Duration::from_secs(30))` — matches TS `REQUEST_TIMEOUT_MS = 30_000`

    Update `src/main.rs`:
    - Add `mod security; mod auth; mod http;` declarations
    - Do NOT wire HttpClient into commands yet (Phase 5 concern) — just ensure modules compile

    The client should be lazily constructable: commands that need it will call `HttpClient::new(backend_url, signer)` when they run (Phase 5). For now, only the building blocks need to exist and be tested.
