# Phase 5 Plan 01: Port All CLI Commands to Rust

## Objective

Port all 10 existing CLI commands (register, deposit, balance, queue, dequeue, poll, submit, status, matches, match) from the TypeScript CLI to Rust, using alloy for on-chain interactions and EIP-712 signing.

## Context

- @src/main.rs — existing stubs for all 10 commands
- @src/http.rs — authenticated HTTP client
- @src/wallet.rs — wallet loading
- @src/config.rs — config resolution
- @src/security.rs — secret leak detection, path sanitization
- @clawduel-cli.ts — TypeScript reference implementation

## Tasks

### Task 1: Create contracts module and add alloy provider dependencies
type="auto"

Create `src/contracts.rs` with:
- Contract address resolution from env vars with defaults
- ABI definitions for USDC (approve, balanceOf), Bank (deposit, balanceOf, lockedBalanceOf), ClawDuel (usedNonces)
- Helper to create alloy provider from RPC URL
- Update Cargo.toml to add alloy provider/contract/network/sol-types features
- Update lib.rs to export contracts module

**Done when:** `cargo build` compiles with the contracts module

### Task 2: Implement simple API commands (register, dequeue, status, matches, match)
type="auto"

Create command files:
- `src/commands/register.rs` — POST /agents/register with {nickname}
- `src/commands/dequeue.rs` — DELETE /duels/queue with {betTier} (USDC amount * 1e6)
- `src/commands/status.rs` — GET /api/agents/{address} + on-chain balance
- `src/commands/matches.rs` — GET /api/matches with query params
- `src/commands/match_detail.rs` — GET /api/matches/{matchId} with resolution summary

Update `src/commands/mod.rs` to export all new modules.

**Done when:** All 5 command modules compile

### Task 3: Implement on-chain commands (deposit, balance)
type="auto"

Create:
- `src/commands/balance.rs` — On-chain bank.balanceOf + bank.lockedBalanceOf, format as USDC (6 decimals)
- `src/commands/deposit.rs` — On-chain USDC approve + bank.deposit, with balance check

**Done when:** Both modules compile

### Task 4: Implement queue command with EIP-712 signing
type="auto"

Create `src/commands/queue.rs`:
- Generate random nonce, verify not used on-chain via clawDuel.usedNonces
- Build EIP-712 domain (ClawDuel, version 1, chainId from provider, verifyingContract)
- Build JoinDuelAttestation typed data (agent, betTier, nonce, deadline)
- Sign with alloy's SignerSync or Signer
- POST /duels/queue with {betTier, signature, nonce, deadline}

**Done when:** Module compiles with EIP-712 signing

### Task 5: Implement poll and submit commands
type="auto"

Create:
- `src/commands/poll.rs` — GET /matches/active/{address}, handle ready acknowledgement (POST readyUrl), waiting_start (sleep until startsAt), re-poll
- `src/commands/submit.rs` — Sanitize prediction text (strip control chars, normalize whitespace), POST /matches/{matchId}/submit

**Done when:** Both modules compile

### Task 6: Wire all commands in main.rs and verify compilation
type="auto"

Update `src/main.rs`:
- Import and wire all command modules
- Each command creates HttpClient and/or provider as needed
- Pass correct args from CLI to command functions
- Wallet commands don't need wallet loading; all others do

Run `cargo build` to verify everything compiles.

**Done when:** `cargo build` succeeds with zero errors

## Verification

- `cargo build` compiles successfully
- All 10 commands are wired in main.rs
- EIP-712 signing is implemented for queue command
- Prediction sanitization is implemented for submit command

## Success Criteria

1. All 10 command modules exist and compile
2. Queue command has EIP-712 attestation signing
3. Submit command sanitizes prediction text
4. `cargo build` passes
