# Phase 6 Plan 01: Output, Shell & Distribution

## Objective
Add dual output format (table/json), interactive shell, status/upgrade commands, and release profile optimization.

## Context
- @src/main.rs (current CLI entrypoint)
- @src/commands/ (all existing command handlers)
- @Cargo.toml (dependencies and profiles)
- @polymarket-cli-ref (architecture reference for output module, shell, upgrade)

## Tasks

### Task 1: Output module + tabled dependency
type="auto"

Add `tabled` and `rustyline` crates to Cargo.toml. Create `src/output.rs` with:
- `OutputFormat` enum (Table, Json) with clap ValueEnum derive
- `print_json` helper
- `print_table` helper using tabled
- Per-command table rendering functions

**Done when:** `src/output.rs` exists with OutputFormat enum and helpers, Cargo.toml has tabled + rustyline deps.

### Task 2: Add --output global flag + wire to all commands
type="auto"

Update main.rs Cli struct with `--output table|json` global flag (default: table). Pass OutputFormat to each command's execute function. Update every command handler to accept OutputFormat and render accordingly:
- Table mode: pretty formatted output (use tabled where appropriate)
- JSON mode: serde_json::to_string_pretty, no println! side effects

**Done when:** All commands accept OutputFormat, `--output json` produces clean JSON, `--output table` produces pretty tables.

### Task 3: Interactive shell
type="auto"

Create `src/shell.rs` with rustyline-based REPL:
- `clawduel shell` launches REPL
- Prompt: "clawduel> "
- Parse input, prepend "clawduel", parse with Cli::try_parse_from
- Handle exit/quit, skip "shell" subcommand in shell mode
- History support

Add Shell variant to Commands enum in main.rs, wire it up.

**Done when:** `clawduel shell` launches interactive REPL, commands work inside shell.

### Task 4: Upgrade command
type="auto"

Create `src/commands/upgrade.rs` with self-update logic:
- Check latest release from GitHub API
- Download and replace binary (or print cargo install instructions as fallback)

**Done when:** `clawduel upgrade` checks for updates and provides upgrade path.

### Task 5: Verify compilation
type="auto"

Run `cargo build` and fix any compilation errors.

**Done when:** `cargo build` succeeds with no errors.

## Verification
- `cargo build` compiles successfully
- All commands accept --output flag

## Success Criteria
1. `--output json` on any command produces machine-parseable JSON
2. `--output table` produces pretty table output
3. `clawduel shell` launches interactive REPL
4. `clawduel upgrade` provides upgrade path
5. Release profile has LTO, strip, codegen-units=1
