3. In `src/commands/queue.rs`, update `execute` signature to accept `games: u64`.

4. When `games <= 1`, behavior is identical to current (queue once, return).

5. When `games > 1`, implement a sequential game loop called `games_loop`:
   - For game_num in 1..=games:
     a. Print `"\n=== Game {game_num}/{games} ==="` in table mode
     b. Run the existing queue logic (EIP-712 sign, POST to /duels/queue)
     c. If queue fails, print error with `"Completed {game_num-1}/{games} games"` and return error
     d. Wait for match assignment: poll `/matches/active/{address}` every 3s until a match with status `waiting_submissions` and a non-null `problem` is found. Use the `poll_once` helper from `src/commands/poll.rs` (make it `pub` if not already). Max wait: 300s.
     e. Once match found, extract match_id. Print match info in table mode.
     f. Wait for resolution: poll `/api/matches/{match_id}` every 10s until `status == "resolved"`. Max wait: 600s. Use a local helper (don't import match_detail to avoid coupling -- just do a simple GET loop).
     g. Once resolved, print a summary line: `"Game {game_num}: {result}"` where result shows winner or draw
     h. If game_num < games, sleep 2s then continue to next iteration

6. In JSON mode, collect all game results into a Vec and emit a single JSON array at the end:
   ```json
   [
     {"game": 1, "matchId": "...", "status": "resolved", "winner": "..."},
     {"game": 2, "matchId": "...", "status": "resolved", "winner": "..."}
   ]
   ```

7. Make `poll::poll_once` public (`pub async fn poll_once(...)`) so queue.rs can call it for the match-waiting phase. If poll_once doesn't exist yet (should from Plan 01), create it.

8. For the resolution wait loop inside queue.rs, write a simple local helper:
   ```rust
   async fn wait_for_resolution(client: &HttpClient, match_id: &str, interval_secs: u64, timeout_secs: u64) -> Result<serde_json::Value> {
       let start = Instant::now();
       loop {
           let safe_id = security::sanitize_path_segment(match_id);
           let data = client.get(&format!("/api/matches/{safe_id}")).await?;
           if data.get("status").and_then(|s| s.as_str()) == Some("resolved") {
               return Ok(data);
           }
           if start.elapsed().as_secs() > timeout_secs {
               anyhow::bail!("Timeout waiting for match resolution after {}s", timeout_secs);
           }
           tokio::time::sleep(Duration::from_secs(interval_secs)).await;
       }
   }
   ```
